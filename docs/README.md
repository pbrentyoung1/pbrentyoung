# The Documentation

Everything in this repository should reinforce one simple belief:

**The right story, told the right way.**

We believe communication is ministry, design solves creative problems, and great creativity is built on great systems. Every page, article, case study, and project should help churches and mission-driven organizations communicate with greater clarity, consistency, and purpose.

## How They Work Together

FIRST_PRINCIPLES defines **why**.

EDITORIAL_GUIDE defines **how it sounds**.

CONTENT_ROADMAP defines **what comes next**.

IDEA_GLOSSARY defines **what words mean**.

ARTICLE_TEMPLATE defines **how an article is built**.

These are living documents. They should evolve as the philosophy matures — but they should evolve deliberately, the way a constitution is amended rather than the way a feed is updated.

## Every Decision Should Answer Four Questions

Who are we?

Who are we trying to reach?

What problem are we solving?

What story will people experience?
